1. Integrasi Auth & Login (Rekomendasi Utama)

Ini yang paling krusial. Kita tes apakah LoginRequest kamu sudah sinkron.

    Target: Siswa/Guru login -> Backend kirim JWT Token & data User -> Frontend simpan di localStorage atau cookies.

    Kenapa ini dulu? Karena semua halaman dashboard (Admin/Guru/Siswa) butuh data user yang asli untuk nampilin nama dan role.

2. Integrasi Manajemen User (Testing CRUD)

Kita tes satu menu dulu, misal Manajemen Siswa.

    Target: Ganti data dummy students = $state([...]) dengan hasil fetch() dari API Go kamu.

    Kenapa? Untuk memastikan struct UserDashboardRow dan SiswaRequest yang kita buat di Svelte sudah "satu frekuensi" dengan JSON yang dikirim Go.

3. Integrasi QR & Scanner (Testing Logic)

Kalau kamu ingin langsung ke "fitur utama".

    Target: Guru generate QR (simpan ke DB) -> Siswa scan -> API Go validasi (Cek device ID, cek lokasi, cek waktu).

    Kenapa? Ini bagian paling kompleks. Semakin cepat kita tes, semakin cepat kita tahu kalau ada logic yang kurang.
